# KrishaAnalyzer Skill

## Описание

KrishaAnalyzer — AI-агент, который анализирует объявления о продаже квартир на Krisha.kz и помогает покупателю принять решение: брать, торговаться или искать дальше.

Агент берёт URL объявления, скрапит данные, сравнивает с рынком из базы 37 000+ объявлений, анализирует фото через GPT-4o Vision и выдаёт обоснованный вердикт с числами.

---

## Триггеры запуска

Агент вызывается, когда пользователь:
- Вставляет ссылку вида `https://krisha.kz/a/show/<id>`
- Хочет узнать, справедлива ли цена на квартиру
- Хочет сравнить объявление с рынком
- Задаёт вопрос о конкретном объявлении

Примеры запросов:
```
https://krisha.kz/a/show/1010382901
Стоит ли брать эту квартиру? https://krisha.kz/a/show/698960276
Проверь цену: krisha.kz/a/show/1010111343
```

## Пайплайн (6 нод LangGraph)

```
fetch_listing
    ↓ (если ошибка → END)
search_similar
    ↓
analyze_photos
    ↓
evaluate_price
    ↓
generate_recommendation
    ↓
human_review ← (follow-up вопросы пользователя)
    ↓
END
```

### Node 1: `fetch_listing`
- Скрапит страницу объявления (BeautifulSoup)
- Извлекает: название, цену, площадь, этаж, район, описание, URL фото
- Парсит год постройки и тип дома из описания (regex)
- Возвращает структурированный `listing_data`

### Node 2: `search_similar`
- RAG-поиск через ChromaDB (36 686 эмбеддингов, `text-embedding-3-small`)
- Запрос: `"N-комнатная квартира <район> <площадь>м²"`
- Дополняет live данными из Krisha API (если доступен)
- Дедупликация по URL или title
- Возвращает top-5 похожих объявлений

### Node 3: `analyze_photos`
- Скачивает все фото из объявления (до 10 штук)
- GPT-4o Vision: оценка состояния квартиры по шкале 1-10
- Учитывает подъезд, стены, пол, потолки, сантехнику
- Детерминированная конвертация score → корректировка цены:
  - 9-10: +10% | 7-8: +3% | 5-6: 0% | 3-4: -8% | 1-2: -15%
- Возвращает: `condition_score`, `renovation_needed`, `issues`, `summary`

### Node 4: `evaluate_price`
- Получает рыночную цену м² из SQLite (приоритет: локальная база → live скрапинг → API)
- Фильтрует похожие объявления по площади ±30%
- Удаляет выбросы методом IQR
- Применяет корректировки: этаж (-5%/-3%), год постройки (от -35% до +8%), тип дома (-10%/+5%), фото
- Вычисляет справедливый диапазон цены (fair_min, fair_max)
- Вердикт: `выгодно` / `рыночная цена` / `дорого`
- Сравнивает с похожими: `лучшая сделка` / `одна из лучших` / `есть варианты лучше`

### Node 5: `generate_recommendation`
- GPT-4o собирает всё в текстовую рекомендацию
- Формат: Вердикт → Почему → Что влияет → Рекомендация по торгу → Итог
- Использует реальные цифры из предыдущих нод

### Node 6: `human_review`
- Ожидает follow-up вопрос от пользователя
- GPT-4o-mini отвечает в контексте уже выполненного анализа
- Поддерживает неограниченное количество вопросов

## Выходные данные

```python
AgentState = {
    "listing_data": {
        "title": str,        # заголовок объявления
        "price": int,        # цена в тенге
        "area": float,       # площадь м²
        "rooms": int,        # количество комнат
        "floor": int,        # этаж
        "floors_total": int, # всего этажей
        "district": str,     # район Алматы
        "price_m2": int,     # цена за м²
        "photo_urls": list,  # ссылки на фото
        "url": str,          # URL объявления
    },
    "similar_listings": list,     # топ-5 похожих объявлений
    "photo_analysis": {
        "condition_score": int,   # оценка 1-10
        "renovation_needed": str, # "не нужен" / "косметический" / "полный"
        "issues": list,           # обнаруженные проблемы
        "summary": str,           # краткое описание состояния
        "price_adjustment_pct": float, # корректировка цены
    },
    "price_stats": {
        "avg_price_m2": int,      # средняя цена м² по рынку
        "median_price_m2": int,   # медианная цена м²
        "sample_size": int,       # объём выборки
        "source": str,            # источник данных
    },
    "price_evaluation": {
        "verdict": str,           # "выгодно" / "рыночная цена" / "дорого"
        "verdict_detail": str,    # детализация вердикта
        "fair_min": int,          # нижняя граница справедливой цены
        "fair_max": int,          # верхняя граница справедливой цены
        "total_adjustment_pct": float, # итоговая корректировка %
    },
    "market_comparison": {
        "verdict": str,           # "лучшая сделка" / "одна из лучших" / "есть варианты лучше"
        "detail": str,
        "best_alternative": dict, # лучшая альтернатива (если есть)
    },
    "recommendation": str,        # текстовая рекомендация агента
    "error": str | None,
}
```

## Используемые инструменты

| Инструмент | Назначение |
|-----------|-----------|
| `search_listings` (MCP) | Поиск объявлений по параметрам через Krisha API |
| `get_price_statistics` (MCP) | Статистика цен м² по рынку |
| `estimate_fair_value` (MCP) | Оценка справедливой стоимости |
| ChromaDB RAG | Поиск похожих объявлений по семантическому сходству |
| SQLite | Локальная база 37 763 объявлений для точной статистики |
| GPT-4o Vision | Анализ фото квартиры |
| GPT-4o | Генерация текстовой рекомендации |
| GPT-4o-mini | Ответы на follow-up вопросы |

---

## Модели и параметры

| Модель | Использование | Temperature |
|--------|--------------|-------------|
| `gpt-4o` | Vision анализ + рекомендация | 0.1 / 0.4 |
| `gpt-4o-mini` | Follow-up вопросы | 0.3 |
| `text-embedding-3-small` | RAG эмбеддинги | — |

**Обоснование температур:**
- `0.1` для анализа фото — нужна строгость, не фантазия
- `0.4` для рекомендации — нужен связный нарратив, допустима небольшая вариативность
- `0.3` для follow-up — баланс точности и естественности

---

## Ограничения

- Работает только с объявлениями Krisha.kz (формат `/a/show/<id>`)
- Статистика цен актуальна для 8 районов Алматы
- Для 5-комнатных квартир точность ниже (малая выборка)
- Live Krisha API периодически блокирует запросы — fallback на локальную базу
- Анализ фото недоступен если у объявления нет фотографий
